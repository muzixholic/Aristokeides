## Layout

### Spacing Scale
Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, kanban card padding |
| md | 16px | Default element spacing, kanban column gaps, card margins |
| lg | 24px | Section padding, kanban board container padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none

### Layout Strategy
- **Kanban Board**: Horizontal scrollable layout (`display: flex; overflow-x: auto; gap: 16px;`). Columns have fixed width (e.g., `300px`) and `flex-shrink: 0`.
- **Forms**: Vertical stack for issue creation and detail views. Container `max-width: 800px; margin: 0 auto;`.

## Components

- **Kanban Column**: A container with a grey background (`var(--secondary)`), containing a header (Column Name) and a vertical list of Issue Cards.
- **Issue Card**: A white card (`var(--dominant)`) with a border, displaying the issue title, Local ID (e.g., #1), and Assignee. Has a `cursor: grab` style.
- **Repo Navigation Tab**: A horizontal navigation bar within the repository view to switch between "Code" and "Issues".
- **Form Inputs**: Standard text inputs and textareas with border and padding using spacing scale (`sm` or `md`), background `var(--dominant)`.

---

## Interaction

- **HTML5 Drag and Drop (Kanban Board)**:
  - `ondragstart` (Issue Card): Sets the drag payload (Issue ID) and visually updates the card (e.g., `opacity: 0.5`).
  - `ondragover` (Column): Prevent default behavior to allow dropping. Visually highlights the drop target column.
  - `ondrop` (Column): Captures the dragged Issue ID, updates its `BoardColumnId`, and triggers a `_db.SaveChangesAsync()`.
  - **Blazor Mode**: All kanban interaction requires `@rendermode InteractiveServer`.
- **Navigation**: Switching between Code and Issues should be smooth, maintaining repository context.
- **Form Validation**: Standard Blazor form handling (EditForm, DataAnnotationsValidator) before submission.
