## Phase Goal

**As a** PR reviewer, **I want to** write inline comments on code lines in a PR diff and reply to threads, **so that** we can review and resolve discussions before merging.

<objective>
이 계획은 PR 파일 변경 Diff 화면에서 특정 코드 라인에 단일 인라인 댓글을 작성하고 이를 데이터베이스에 영속화하며, 새로고침 없이 화면에 즉각 렌더링하는 핵심 수직 슬라이스(Vertical Slice)를 구축합니다.
목적: 코드 리뷰어가 특정 라인 단위로 피드백을 남길 수 있는 핵심 데이터 파이프라인 및 기본 UI 연동을 제공합니다.
산출물: PullRequestReviewComment 엔터티, DiffParser 유틸리티, PullRequestService 댓글 CRUD 비즈니스 로직, RepoPullRequestDetail.razor 기본 인라인 댓글 연동.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing / Elevation of Privilege | PullRequestService | mitigate | 댓글 추가/조회 시 로그인된 사용자 세션을 확인하고, 사용자가 대상 리포지토리에 대한 접근 권한(D-02 기반 역할 검증)을 보유했는지 필수로 체크함. |
| T-08-02 | Tampering | PullRequestService / DiffParser | mitigate | 클라이언트로부터 임의로 유입되는 `OldLineNumber`, `NewLineNumber`, `FilePath` 등이 실제 PR Diff 범위 내에 존재하는지 비즈니스 로직단에서 교차 검증을 거친 후 DB에 저장함. |
