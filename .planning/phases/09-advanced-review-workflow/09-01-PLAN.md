## Phase Goal

**As a** repository contributor and maintainer, **I want to** manage code review statuses (Comment, Approve, Request Changes), submit multiple inline comments in a batch, preserve existing comments through Line Shifts during pushes, and enforce merge safety checks, **so that** code quality is fully audited and unsafe merges are blocked.

<objective>
이 계획은 고급 리뷰 워크플로우를 구성하기 위한 백엔드 데이터 모델, 마이그레이션, 비즈니스 서비스 로직 및 Git Push 이벤트 기반 후처리 로직(라인 보정 및 승인 초기화)을 통합 구현합니다.
목적: 임시 보관 리뷰 메커니즘, 라인 보정 알고리즘, 미해결 토론 병합 차단, 브랜치 푸시 후처리 등 백엔드 핵심 비즈니스 기능을 안정적으로 제공합니다.
산출물: 신규 `PullRequestReview` 모델, 데이터베이스 스키마 확장, `PullRequestService`에 대한 로직 구현(리뷰 제출, 머지 검증, 푸시 후처리), HTTP/SSH Git 엔드포인트 연동.
</objective>

<task type="auto">
  <name>Task 3: Git Push 후처리 로직 (Line Shift, Outdated 전환, 승인 초기화) 구현 및 통합</name>
  <files>Aristokeides.Api/Services/PullRequestService.cs, Aristokeides.Api/Middleware/GitSmartHttpMiddleware.cs, Aristokeides.Api/Services/Ssh/SshCommandBridge.cs, Aristokeides.Tests/Services/AdvancedReviewTests.cs</files>
  <action>
    - `PullRequestService.cs`에 `OnBranchPushedAsync(Guid repositoryId, string branchName, string oldOid, string newOid)` 메소드를 작성합니다.
      - 이 브랜치가 소스 브랜치(`SourceBranch == branchName`)이고 아직 병합되지 않은 모든 PR 목록을 DB에서 조회합니다.
      - `oldOid`와 `newOid` 사이의 Git Diff를 생성하여 `DiffParser.Parse`로 변경된 파일의 Hunk 정보를 수집합니다.
      - 변경된 각 파일에 대해:
        - 해당 파일에 매핑된 해결되지 않은 기존 댓글들을 가져옵니다.
        - Myers/Hunk 매핑 보정 알고리즘을 사용해:
          - 기존 댓글의 줄 위치(`NewLineNumber` 혹은 `OldLineNumber`)가 신규 푸시의 변경 Hunk 범위보다 앞이면 영향 없음.
          - 변경 Hunk 범위 내에 있으면, 해당 줄이 삭제/수정되었을 경우 `IsOutdated = true`로 설정합니다. 만약 새 버전에서도 유지된다면 새 라인 번호로 보정합니다.
          - 변경 Hunk 범위보다 뒤에 있으면, 변경량(Shift Delta = newLen - oldLen) 만큼 라인 번호를 이동(Line Shift)해 줍니다.
      - 새로운 커밋이 푸시되었으므로, 해당 PR에 존재하던 기존 리뷰어들의 모든 `State == PullRequestReviewState.Approved`인 리뷰들을 `State = PullRequestReviewState.Dismissed` 상태로 일괄 리셋(초기화)합니다.
      - 변경된 사항들을 DB에 반영합니다.
    - `GitSmartHttpMiddleware.cs`의 HTTP Push 완료 후처리 블록에서 `VerifyNewCommitsAsync` 실행과 함께 `OnBranchPushedAsync`를 비동기(`Task.Run`)로 호출하도록 연동합니다. (r.CanonicalName에서 refs/heads/를 파싱하여 브랜치명 추출)
    - `SshCommandBridge.cs`의 SSH Push 완료 후처리 블록에서도 동일하게 `OnBranchPushedAsync`를 비동기로 호출하도록 연동합니다.
    - `AdvancedReviewTests.cs`에 Line Shift 알고리즘 및 승인 리셋 작동 여부를 검증하는 통합 단위 테스트를 작성하고 실행합니다.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Elevation of Privilege | PullRequestService.MergePullRequestAsync | mitigate | 병합 시 미해결 토론이 있어 강제 병합(`forceMerge = true`)을 요청하는 경우, 백엔드 단에서 `currentUserId`에 해당하는 User의 Role이 `Admin`인지 직접 검증하여 일반 사용자의 권한 남용을 원천 차단함. |
