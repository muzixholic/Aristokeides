## Phase Goal

**As a** PR reviewer and manager, **I want to** use an interactive web interface to start reviews, batch submit comments, visually distinguish outdated comments, and enforce or bypass merge blocks based on unresolved threads, **so that** the code review workflow is intuitive and secure.

<objective>
이 계획은 Wave 1에서 개발된 백엔드 비즈니스 로직을 Blazor Server SSR 대화형 UI에 완벽히 결합합니다.
목적: 사용자에게 편리한 리뷰 임시 보관 및 일괄 제출 인터페이스, Outdated 댓글에 대한 명확한 시각적 피드백, 그리고 병합 안정성을 보장하면서도 관리자에게 유연성을 제공하는 UI 환경을 완성합니다.
산출물: 개정된 `RepoPullRequestDetail.razor` UI.
</objective>

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Blazor UI 댓글 일괄 제출(Submit review) 및 Pending 상태 제어 구현</name>
  <files>Aristokeides.Api/Components/Pages/RepoPullRequestDetail.razor</files>
  <action>
    - `RepoPullRequestDetail.razor` 의 인라인 댓글 에디터 하단 버튼 영역에 "Add single comment" 버튼과 함께 "Start a review" (또는 이미 리뷰가 시작되었다면 "Add review comment") 버튼을 추가합니다.
      - "Start a review" 클릭 시 `IsPending = true`로 댓글을 생성하도록 호출합니다.
    - 사용자가 임시 보관 중인 댓글 목록을 UI 렌더링 시 필터링하여, Pending 상태의 댓글 옆에는 연한 주황색의 "Pending" 배지를 시각적으로 표시합니다.
    - 현재 사용자가 작성한 Pending 댓글이 1개 이상 존재할 때, PR 상세 페이지 상단 또는 Diff 뷰 상단 영역에 "Finish your review" 혹은 "Submit review" 형태의 플로팅 배너/조작 영역을 렌더링합니다.
    - "Submit review" 버튼 클릭 시 리뷰 총평(Body) 입력용 텍스트 필드와 라디오 버튼 그룹(Comment, Approve, Request Changes)을 포함하는 제출 패널을 토글 노출합니다.
    - 제출 패널에서 "Submit" 클릭 시 백엔드 `SubmitReviewAsync` 메소드를 실행하고 UI 데이터를 새로고침(`LoadReviewComments`)하여 모든 댓글의 Pending 배지가 사라지고 최종 리뷰 요약 글이 렌더링되게 합니다.
  </action>
  <verify>
    <human-check>
      1. 로컬에서 실행 후 PR Diff 창에서 특정 코드 라인의 "+"를 누르고 "Start a review"로 댓글을 작성합니다.
      2. 작성된 댓글 옆에 "Pending" 배지가 표시되는지 확인합니다.
      3. 화면에 "Submit review" 버튼/패널이 나타나는지 확인합니다.
      4. 패널에서 "Approve"를 선택하고 의견을 작성해 제출한 뒤, Pending 배지가 사라지고 사이드바 혹은 히스토리에 승인 이력이 정상 노출되는지 검증합니다.
    </human-check>
  </verify>
  <done>
    임시 보관 및 일괄 리뷰 제출 UI 연동이 자연스럽고 프리미엄 스타일 가이드에 맞게 구동됨.
  </done>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: UI 병합 제어(Merge Block) 및 관리자 우회(Force Merge) 옵션 구현</name>
  <files>Aristokeides.Api/Components/Pages/RepoPullRequestDetail.razor</files>
  <action>
    - `RepoPullRequestDetail.razor` 내에서 PR에 해결되지 않은(`IsResolved == false` 및 `IsPending == false`) 스레드 개수를 계산합니다.
    - 사이드바의 병합 제어 영역을 수정합니다:
      - 미해결 스레드가 존재할 경우:
        - 일반 사용자의 경우 병합 버튼을 완전히 비활성화(`disabled`) 처리하고, "Unresolved conversations must be resolved before merging."이라는 경고 배너를 노출합니다.
        - 관리자(User.Role == "Admin")인 경우: "There are unresolved conversations, but I want to force merge." 라는 체크박스를 노출합니다. 이 체크박스를 체크한 경우에만 "Force merge pull request" 버튼을 활성화합니다.
      - 병합 버튼을 누르면 `MergePullRequestAsync`를 호출할 때 관리자가 체크박스를 체크했다면 `forceMerge = true`를 매개변수로 함께 전달하여 실행합니다.
  </action>
  <verify>
    <human-check>
      1. 해결되지 않은 인라인 댓글이 있는 상태에서 일반 사용자 계정으로 로그인하여 PR 상세 페이지를 봅니다.
      2. 병합(Merge) 버튼이 비활성화 상태이고 경고 메시지가 나타나는지 검증합니다.
      3. 관리자 계정으로 전환하여 확인했을 때 강제 병합 체크박스가 노출되는지 확인합니다.
      4. 체크박스를 해제하면 머지 버튼이 비활성화되고, 체크하면 "Force merge pull request" 버튼이 활성화되어 실제 강제 병합이 성공하는지 검증합니다.
    </human-check>
  </verify>
  <done>
    미해결 토론 병합 제한 및 관리자용 강제 병합 우회 처리 UI 연동이 안전장치와 함께 완벽히 수행됨.
  </done>
</task>

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 3: UI Outdated 댓글 접기 및 시각적 개선</name>
  <files>Aristokeides.Api/Components/Pages/RepoPullRequestDetail.razor</files>
  <action>
    - `IsOutdated == true` 상태의 댓글 스레드는 Diff 테이블 내에서 기본적으로 내용을 접어 숨깁니다.
    - 해당 댓글 라인에는 "Outdated conversation (Click to expand)"과 같은 회색 슬림 배너를 노출하고, 클릭 시 토글하여 기존의 전체 댓글 히스토리를 조회할 수 있도록 블레이저 렌더링 조건문을 보완합니다.
    - 새로운 푸시로 인해 승인이 취소(`Dismissed` 상태)된 경우, 리뷰 히스토리 타임라인 상에 "Approved status was dismissed because of new commits" 형태의 시스템 알림 메시지를 시각적으로 보기 좋게 렌더링합니다.
  </action>
  <verify>
    <human-check>
      1. 이미 댓글이 달린 소스 브랜치에 로컬 터미널에서 신규 커밋을 작성해 Push합니다.
      2. 브라우저로 PR 상세 페이지를 새로고침하여, 해당 라인의 기존 댓글이 "Outdated conversation" 배너로 접혀 있는지 확인하고 클릭 시 정상적으로 펼쳐지는지 확인합니다.
      3. 이전에 승인했던 리뷰가 타임라인에 취소(Dismissed) 이력으로 정상 표기되는지 검증합니다.
    </human-check>
  </verify>
  <done>
    Outdated 스레드의 접기 연동 및 승인 취소 이력 타임라인 렌더링이 가이드라인에 부합하게 구현됨.
  </done>
