## Phase Goal

**As a** registered user, **I want to** register and delete my SSH public keys on the settings page, **so that** I can configure my public keys for secure SSH-based Git operations.

<objective>
이 계획은 사용자가 SSH 공개키를 시스템에 안전하게 등록하고 삭제할 수 있는 기반 기능(DB 엔티티, 파서, 마이그레이션, API 컨트롤러, Blazor UI 설정 탭)을 수직 슬라이스로 구축하는 것을 목표로 합니다.

목적: SSH 기반 인증을 처리하기 위해 공개키의 알고리즘 정합성 검증 및 전역 중복 방지를 보장하면서 사용자 키 관리 기능을 제공합니다.
산출물: SshKey 모델, DB 마이그레이션, SshKeyParser, SshFingerprintCalculator, SshKeysController, Settings.razor 설정 탭
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Web UI → SshKeysController | 클라이언트가 전송한 임의의 SSH 공개키 텍스트가 컨트롤러로 전달되는 경계 |
| Controller → Database | 추출된 Fingerprint 중복 체크 및 SshKey 적재를 수행하는 영속성 경계 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | SshKeysController (POST) | mitigate | 동일한 공개키가 타인에 의해 중복 사용되지 않도록 DB 레벨에서 Fingerprint Unique Index를 생성하고 API 레벨에서 409 중복 검출 처리를 수행함. |
| T-06-02 | Elevation of Privilege | SshKeyParser / SshKey | mitigate | RSA-2048 등 암호학적으로 취약한 구형 공개키 등록을 원천 차단하고 RSA 3072비트 이상, ECDSA, Ed25519 알고리즘만 통과시키는 크기/유형 화이트리스트 로직을 Parser에 엄격히 내장함. |
| T-06-03 | Information Disclosure | SshKeysController | mitigate | 잘못된 공개키 양식이 제공되었을 때 원시 스택 트레이스나 구체적인 파서 구조를 그대로 리턴하지 않고 정제된 에러 카피라이팅 메시지만 응답하도록 try-catch 핸들링 구현. |
</threat_model>
