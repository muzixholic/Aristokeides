## Phase Goal

**As a** registered user, **I want to** get the SSH Clone URL on the repository page and test my connection using `ssh -T`, **so that** I can verify that my SSH configuration is working before performing Git actions.

<objective>
이 계획은 저장소 웹 UI에 SSH Clone URL 위젯을 제공하고, FxSsh 기반의 임베디드 SSH 서버를 백그라운드 서비스로 기동하여 사용자의 `ssh -T` 인증 검증 및 진단 웰컴 응답 프로세스를 완성하는 것을 목표로 합니다.

목적: Git 클라이언트가 서버에 SSH 연결을 수립할 수 있도록 하고, DB 데이터와 FxSsh 인증을 동기화하여 진단 도구를 연동합니다.
산출물: RepoBrowser.razor (URL 컴포넌트), SshServerBackgroundService.cs, RepositoryUrlTests, SshTDiagnosticTests
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSH Client → FxSsh Daemon | TCP 2222 포트를 통해 비인증 클라이언트가 SSHv2 핸드셰이크를 요청하는 경계 |
| SSH Key Verify Packet → AppDbContext | 수집된 클라이언트 SSH Key 패킷 데이터가 사용자 조회를 위해 데이터베이스 쿼리를 실행하는 경계 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Spoofing | SshServerBackgroundService (UserAuth) | mitigate | FxSsh 이벤트 내의 SignatureVerified 속성을 반드시 판별하고, DB 조회 지문 일치 여부를 & 연산하여 비어 있는 비밀키 서명이나 검증되지 않은 인증 우회 패킷의 통과를 전면 차단함. |
| T-06-05 | Spoofing | SshServerBackgroundService (Host Key) | mitigate | 매 구동마다 호스트 키를 재생성하면 MITM 공격 경고가 로컬 클라이언트에 출력되어 신뢰도를 해치므로, 최초 실행 시 'host.key' 파일로 디스크에 고정 보관 및 매회 재사용하도록 완화함. |
| T-06-SC | Tampering | NuGet Packages (FxSsh) | mitigate | FxSsh 1.3.0 버전에 대해 NuGet 공식 서명 및 취약점 이력 조회를 마쳐 승인되었음을 확정하고, blocking human checkpoint 없이 로드함. |
</threat_model>
