## Phase Goal

**As a** developer using local Git client, **I want to** run standard Git commands (clone, push, pull) via SSH protocol without full shell access, **so that** I can transfer code securely under strict authorization.

<objective>
이 계획은 비인가 셸 접속을 방어하고, OS Git 프로세스(`git-upload-pack` 및 `git-receive-pack`)의 표준 입출력 스트림을 FxSsh 채널과 비동기적으로 중계 연동함으로써 로컬 Git 명령(Clone, Push, Pull)을 완전히 완수하는 것을 목표로 합니다.

목적: 셸 인젝션 등 취약점을 제거하고 프로세스 자원 누수 없이 안정적인 Git SSH 수송 계층을 연동합니다.
산출물: SshCommandBridge.cs, SshCommandPipingTests.cs, SshServerAuthTests.cs
</objective>

    - OS의 git 바이너리를 실행하는 ProcessStartInfo를 선언하며, 셸 인젝션 방지를 위해 UseShellExecute = false, CreateNoWindow = true를 엄격히 지정하고 RedirectStandardInput/Output/Error를 true로 지정합니다.
    - FxSsh 채널의 DataReceived 이벤트를 바인딩해 수신된 바이트 어레이를 process.StandardInput.BaseStream에 다이렉트로 Write 및 Flush하도록 파이핑합니다.

    - process.WaitForExitAsync를 대기하고 종료된 경우 StandardInput 스트림을 명시적으로 클로즈하며, 스트림 리딩 태스크들이 모두 완료된 것을 Task.WhenAll로 수거한 뒤 channel.SendExitStatus(process.ExitCode) 및 channel.SendClose()를 try-finally 블록에서 무조건 보장 실행하도록 구현해 세션 Hanging 오류를 차단합니다.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CommandOpened Parameter → Process.Start | 클라이언트로부터 공급받은 명령어 파라미터가 OS 서브프로세스로 전달되는 경계 |
| OS Git StdIn/StdOut → SSH Channel | OS 자식 프로세스의 I/O 바이트 스트림이 SSH 소켓 수송 계층으로 상호 복사되는 경계 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Tampering / Elevation of Privilege | SshCommandBridge | mitigate | CLI command injection 위협을 완화하기 위해 OS 셸 인터프리터(cmd.exe / bash)를 통하지 않는 UseShellExecute = false를 구성하며, FileName = "git"에 인자를 분리 인스트럭션으로 매핑 호출함. |
| T-06-07 | Information Disclosure | SshServerBackgroundService (Path Verification) | mitigate | 사용자가 .. 경로를 사용하여 상위 폴더의 다른 Git 파일이나 기밀 폴더를 clone할 수 없도록, 경로 문자열 검사 단계에서 Directory Traversal 유입을 강제 필터링함. |
| T-06-08 | Denial of Service | SshCommandBridge (Resource Leak) | mitigate | Git 전송이 강제 단절되거나 프로세스 완료 후 스트림 누수로 스레드/서브프로세스가 좀비로 남는 것을 방지하기 위해, try-finally에 StandardInput.Close()와 channel.SendClose()를 강제 보장함. |
</threat_model>
