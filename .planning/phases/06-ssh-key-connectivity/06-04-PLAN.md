## Phase Goal

**사용자(개발자)로서** 로컬 Git 클라이언트에서 커스텀 SSH 키를 통해 연결을 시도할 때, **올바른 키를 제출하지 못해 발생하는 불필요한 패스워드 프롬프트를 방지하고 원활히 인증을 완료하고 싶다.** 이를 위해 **웹 UI 상에서 구체적인 클라이언트 SSH 설정 방법과 문제 해결 가이드를 명확히 확인하고자 한다.**

<objective>
본 계획은 UAT 검증에서 발생한 Test 8, 9, 11의 실패(클라이언트 환경에 따라 SSH 연결 시 비밀번호 입력을 요구하여 최종 Permission denied가 발생하는 현상)를 해결하기 위한 Gap Closure 계획입니다.
서버 측 FxSsh 라이브러리의 프로토콜 특징상 인증 실패 시 타 인증 수단(Password)을 안내하게 되므로, 클라이언트 측에서 명시적으로 올바른 SSH 키를 서버에 제시하도록 유도해야 합니다. 이를 위해 웹 UI의 사용자 가이드를 대폭 강화합니다.
</objective>

<task type="auto">
  <name>Task 1: Settings.razor UI의 SSH 설정 가이드 배너 보완</name>
  <files>Aristokeides.Api/Components/Pages/Settings.razor</files>
  <action>
    - `Settings.razor` 내의 `💡 SSH 설정 팁` 영역을 개선합니다.
    - 사용자가 자신의 SSH 키 목록에 커스텀 키를 등록한 후, 로컬 PC에서 이를 정상적으로 매핑할 수 있도록 구체적인 `~/.ssh/config` 설정 예시를 추가합니다.
    - 특히, 다른 키들과의 혼선을 예방하기 위해 `IdentitiesOnly yes` 설정을 포함하도록 가이드를 보완합니다.
    - 진단 연결 테스트 시 키를 직접 지정할 수 있는 명시적 `ssh -i` 명령어 가이드도 함께 기재합니다.
    - 설정 템플릿 예시:
      ```text
      Host localhost
        HostName localhost
        Port 2222
        User git
        IdentityFile ~/.ssh/id_ed25519  # 실제 키 경로로 수정
        IdentitiesOnly yes
      ```
  </action>
  <verify>
    - 브라우저를 통해 `/settings` 페이지의 SSH 탭에 접속하여 수정된 가이드 배너가 올바르게 렌더링되는지 확인합니다.
  </verify>
  <done>사용자 설정(Settings.razor)의 SSH 키 탭에서 플랫폼별 config 설정 템플릿과 명시적 키 매핑 안내가 깔끔한 마크다운/HTML 형태로 노출됩니다.</done>
</task>

    - 이미 코드가 존재할 때 노출되는 우측 상단 Clone URL 영역 하단의 팁 메시지도 동일하게 `IdentitiesOnly=yes` 환경변수 지정을 제안하도록 업데이트합니다.
  </action>

  </verify>
  <done>저장소 브라우저에서 SSH 프로토콜 선택 시 커스텀 키 매핑을 위한 환경변수 설정 가이드가 강화되어 표시됩니다.</done>
