## 💬 Discussion History (토론 기록)

### 1. SSH 포트 설정 및 Clone URL 형식
- **질문:** SSH 포트 설정 및 Clone URL 형식을 어떻게 구성하시겠습니까?
- **제시된 옵션:**
  1. (선택됨) 기본 포트 2222 지정 (설정 파일로 포트 가변화) 및 ssh:// 형식의 명시적 Clone URL 제공 (`ssh://git@domain:2222/user/repo.git`)
  2. 표준 22번 포트 바인딩 고수 (바인딩 실패 시 구동 오류 처리 및 호스트 OS 권한 요구)
- **사용자 결정:** 비표준 포트 2222번을 기본으로 설정 파일 연동형으로 구축하고, Clone URL도 이에 맞춤.

### 2. SSH 키 등록 시 이름(Label) 부여 방식
- **질문:** SSH 키 등록 시 이름(Label) 부여 방식을 어떻게 구성하시겠습니까?
- **제시된 옵션:**
  1. (선택됨) 키 주석(Comment) 자동 파싱하여 기본 이름으로 제안하되, 사용자가 직접 수정/입력할 수 있도록 허용
  2. 파싱 없이 무조건 사용자가 고유한 이름을 직접 지정하도록 필드로 요구
- **사용자 결정:** 편의성을 높이기 위해 공개키 주석 자동 제안 기능을 적용하되 수정을 허용함.

### 3. 'ssh -T git@domain' 인증 성공 시 터미널 피드백
- **질문:** 'ssh -T git@domain' 인증 성공 시 터미널 피드백을 어떻게 구성하시겠습니까?
- **제시된 옵션:**
  1. (선택됨) 표준 환영 메시지만 간단히 응답 (`Hi [Username]! You've successfully authenticated...`)
  2. 사용자 권한, 등록된 키 요약 등 진단 정보를 추가한 상세 메시지 응답
- **사용자 결정:** 표준적인 심플한 환영 문구와 쉘 불가 안내 메시지 반환 적용.

### 4. Blazor UI 상에서 SSH 키 관리 화면의 위치
- **질문:** Blazor UI 상에서 SSH 키 관리 화면의 위치를 어떻게 구성하시겠습니까?
- **제시된 옵션:**
  1. (선택됨) 기존 프로필/계정 설정 화면 내부에 'SSH 키 관리' 탭으로 인라인 추가 (컴포넌트 재사용)
  2. 설정 메뉴 하위에 완전히 독립된 전용 설정 페이지(`/settings/ssh`)로 분리 구성
- **사용자 결정:** 기존 설정 화면의 신규 탭으로 추가하여 UI의 파편화를 막고 재사용성 강화.
