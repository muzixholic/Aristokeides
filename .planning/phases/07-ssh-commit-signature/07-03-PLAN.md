# Plan 07-03: SSH Commit Signature - UI Layer & Verification

이 계획은 커밋 조회 API 및 DTO를 확장하여 데이터베이스의 서명 검증 정보를 통합 제공하고, Blazor Server 웹 UI의 커밋 히스토리 및 상세 페이지에 신뢰할 수 있는 커밋을 뜻하는 연두 초록색 "Verified" 배지와 상세 팝오버 메타데이터를 시각화합니다.

---

## Tasks

### 1. 커밋 조회 DTO 및 GitBrowserService 확장
<read_first>
- Aristokeides.Api/Services/GitBrowserService.cs
</read_first>
<action>
1. `Aristokeides.Api/Services/GitBrowserService.cs`에 정의된 `GitCommitInfo` 레코드를 확장합니다:
   ```csharp
   public record GitCommitInfo(
       string Hash, 
       string Message, 
       string Author, 
       DateTimeOffset Date,
       string? SignatureStatus = null,       // Verified, Invalid, Unknown, NoSignature
       string? SignatureFingerprint = null,  // SSH 키 지문
       string? SignerUsername = null        // 서명 사용자명
   );
   ```
2. `GitBrowserService.GetCommits()` 메서드를 수정합니다.
   - Paged 커밋 리스트(`pagedCommits`)를 조회한 후, 해당 커밋 해시들(`Hash` 값들의 리스트)에 대해 `CommitSignature` 테이블에서 Left Join 처리를 수행하여 검증 정보를 가져옵니다.
   - EF Core를 사용해 커밋 해시와 매칭되는 `CommitSignature` 엔티티 정보를 가져올 때, `SignerUser` 정보도 `Include` 하여 서명자의 사용자명(`SignerUser.Username`)을 함께 가져옵니다.
   - 조회된 DB 데이터를 캐싱/딕셔너리 맵으로 매핑하여 `GitCommitInfo` 빌드 시점에 주입해 줍니다.
</action>
<acceptance_criteria>
- `GitCommitInfo` 레코드 확장이 정상 컴파일됨.
- `GetCommits` 메서드가 호출될 때 각 커밋 해시별로 DB에 연동된 서명 상태, 지문, 서명자 이름 정보가 결합되어 누출 없이 리스트로 반환됨.
</acceptance_criteria>
