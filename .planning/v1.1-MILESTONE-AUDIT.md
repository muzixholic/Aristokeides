## 3. E2E 플로우 검증

### ✅ Flow 1 — 리뷰 일괄 제출
```
"Start a review" → AddReviewCommentAsync(isPending:true) → DB IsPending=true
  → UI Pending 배너 노출 → "Submit review" → SubmitReviewAsync()
  → IsPending=false 일괄 변환 + PullRequestReview 생성 → UI 리렌더링
```
**검증:** `AdvancedReviewTests.SubmitReviewAsync_ActivatesPendingComments_AndCreatesReview`

### ✅ Flow 2 — 병합 차단 / 관리자 우회
```
Unresolved 스레드 존재 → MergePullRequestAsync() → InvalidOperationException
  → UI "Merge Blocked" + Merge 버튼 비활성화
  → Admin 체크박스 → forceMerge=true → 병합 우회 성공
```
**검증:** `AdvancedReviewTests.MergePullRequestAsync_BlocksMerge_WithUnresolvedConversation`

### ✅ Flow 3 — Line Shift / Outdated 전환
```
새 커밋 Push → GitSmartHttpMiddleware/SshCommandBridge 감지
  → OnBranchPushedAsync() → Myers/Hunk 알고리즘
  → 라인 보정(Line Shift) 또는 IsOutdated=true
  → Approved 리뷰 → Dismissed 전환 → UI Outdated 아코디언 + 히스토리
```
**검증:** `AdvancedReviewTests.OnBranchPushedAsync_AppliesLineShift_AndOutdatedState_AndResetsApprovals`

### ✅ Flow 4 — SSH Verified 배지
```
SSH 서명 커밋 Push → SshCommandBridge 완료 → VerifyNewCommitsAsync()
  → git cat-file gpgsig 추출 → SshSignatureParser → Fingerprint
  → db.SshKeys 매칭 → SshSignatureVerifier.Verify() → "Verified" DB Upsert
  → RepoCommits.razor Left Join → 연두색 Verified 배지 + Fingerprint 툴팁
```
**검증:** `PushHookIntegrationTests`, `UiRenderIntegrationTests`
