# Requirements: v1.5 UI 테스트 자동화 및 SSH 호환성 개선

## 1. UI 테스트 자동화 (UI Test Automation)

- [ ] **bUnit 기반 Blazor 컴포넌트 단위 테스트:**
  - [ ] `Aristokeides.Tests` 프로젝트 내에 `bUnit` 의존성 패키지 추가 및 테스트 헬퍼 구성.
  - [ ] 개별 공통 UI 컴포넌트(네비게이션 링크, 버튼 등) 렌더링 검증.
  - [ ] 대화형 UI 동작(로그인 에러 메시지 팝업, 모달 노출 등)의 이벤트 핸들러 및 바인딩 상태 검증.
- [ ] **Playwright for .NET 기반 E2E 브라우저 테스트:**
  - [ ] `Playwright` 테스트 러너 패키지 설치 및 headless/headed 브라우저 구동 스크립트 작성.
  - [ ] 테스트 실행 전 독립된 테스트용 데이터베이스(SQLite 또는 메모리 DB) 프로비저닝 및 시드 데이터 준비 자동화.
- [ ] **핵심 시나리오 E2E 테스트 커버리지:**
  - [ ] **인증/세션**: 웹 기반 회원가입, 로그인, 로그아웃, 그리고 미인가 페이지 접근 제한 흐름 검증.
  - [ ] **저장소 관리**: 웹 기반 Git 저장소 생성, 설정 수정, 그리고 저장소 삭제 흐름 검증.
  - [ ] **협업 기능(PR 및 리뷰)**: 풀 리퀘스트 생성, 커밋 정보 탐색, 인라인 마크다운 댓글 작성 및 리뷰 제출 흐름 검증.
  - [ ] **프로젝트 관리**: 이슈 트래커 생성 및 칸반 보드 내 카드 드래그/상태 변경 동작 흐름 검증.

## 2. SSH 서버 호환성 개선 (SSH Server Compatibility & Replacement)

- [ ] **보안 알고리즘 확장 및 호환성 분석:**
  - [ ] 현대적인 SSH 클라이언트가 표준으로 사용하는 `ed25519` 및 `rsa-sha2-256`, `rsa-sha2-512` 호스트 키/알고리즘 지원 스펙 도출.
  - [ ] 기존 FxSsh 기반의 인증 흐름 및 호스트 키 포맷(PEM) 주입 방식 분석.
- [ ] **SSH 서버 라이브러리 교체 또는 FxSsh 커스텀 패치:**
  - [ ] **방안 A**: FxSsh의 패킷 처리 및 서명 검증 모듈을 직접 패치하여 `ed25519`와 `rsa-sha2` 알고리즘 분석부를 프로젝트 내부에 구현.
  - [ ] **방안 B**: Microsoft의 공식 오픈소스인 `Microsoft.DevTunnels.Ssh`로의 점진적 마이그레이션 타당성 검토 및 파일럿 구현.
  - [ ] 최종 선정된 대안을 백그라운드 서비스(`SshServerBackgroundService`)에 반영하여 무중단 서비스 기동 보장.
- [ ] **SSH 수송 연동성 검증:**
  - [ ] 터미널 환경에서 다양한 SSH 키(ED25519, RSA 등)를 사용한 `ssh -T git@localhost` 진단 접속 정상 통과 보장.
  - [ ] Git 클라이언트(Linux, Windows, MacOS) 환경에서 SSH 프로토콜 기반의 리포지토리 복제(Clone), 푸시(Push), 풀(Pull) 등 핵심 명령어가 `SshCommandBridge`를 통해 안정적으로 양방향 스트림 릴레이가 완료되는지 최종 검증.
