# Requirements (Milestone v1.1)

이 문서는 **Milestone v1.1 (SSH & Advanced Code Review)**에 해당하는 세부 기능 요구사항을 기술합니다. 모든 요구사항은 고유 ID를 가지며 구현 및 검증 단계에서 추적될 수 있습니다.

---

## 🔒 SSH Git 지원 (SSH)

### 1. SSH 키 관리
- [ ] **SSH-01**: 사용자는 프로필 설정 화면에서 자신의 SSH 공개키(`id_rsa.pub`, `id_ed25519.pub` 등)를 등록하고 삭제할 수 있다. (Ed25519, ECDSA, RSA-3072+ 알고리즘만 허용)
- [ ] **SSH-02**: 시스템은 등록 시 동일한 공개키의 시스템 전역 중복 등록을 차단하며, 공개키의 SHA-256 지문(Fingerprint)을 자동 계산하여 웹 UI에 표시한다.

### 2. SSH 프로토콜 지원 및 UX
- [ ] **SSH-03**: 사용자는 저장소 홈 페이지에서 SSH 프로토콜 형식의 Clone URL을 조회하고 원클릭 복사할 수 있다.
- [ ] **SSH-04**: 사용자는 로컬 터미널에서 `ssh -T git@domain` 명령을 통해 SSH 연결 검증을 테스트할 수 있으며, 성공 시 인증 사용자 이름이 담긴 진단 환영 메시지를 응답받는다.
- [ ] **SSH-05**: 시스템은 .NET 호스트 내에서 경량 임베디드 SSH 서버(FxSsh 기반)를 구동하여, DB에 저장된 SSH 공개키와 사용자를 매핑 및 인증한다.
- [ ] **SSH-06**: 시스템은 SSH 세션 연결 성공 시 일반 쉘(Interactive Shell) 실행은 절대 차단하며, 오직 Git 관련 프로세스 명령어(`git-upload-pack`, `git-receive-pack`)의 입출력 중계(파이핑)만 안전하게 허용한다.

### 3. 암호학적 신뢰성 (차별화 요소)
- [ ] **SSH-07**: 시스템은 사용자가 등록한 SSH 키로 서명하여 푸시한 Git 커밋의 디지털 서명을 서버 측에서 검증하고, 웹 UI 커밋 히스토리에 신뢰할 수 있는 커밋 배지("Verified")를 표시한다.

## 📝 라인 단위 코드 리뷰 (CODE)

### 1. PR Diff 인라인 코멘트
- [ ] **CODE-04**: 사용자는 Pull Request의 파일 변경 Diff 화면에서 특정 코드 행에 마우스를 올릴 때 노출되는 `+` 버튼을 통해 해당 줄 바로 아래에 인라인 댓글을 작성(Markdown 및 실시간 프리뷰 지원)할 수 있다.
- [ ] **CODE-05**: 사용자는 단일 인라인 댓글을 즉시 추가하거나, "리뷰 시작"을 통해 여러 댓글을 임시 보관(Pending)했다가 검토 완료 시점에 일괄 제출(Submit review)할 수 있다.
- [ ] **CODE-06**: 시스템은 댓글이 달린 소스 코드 주변 컨텍스트(`DiffHunk`), 원본/대상 파일 경로 및 라인 번호를 데이터베이스에 연동 저장하여 정확한 위치 매칭을 보장한다.
- [ ] **CODE-07**: 시스템은 새 커밋이 브랜치에 푸시되는 경우, Diff 비교를 통해 기존 코멘트의 위치를 보정(Line Shift)하며, 코멘트 영역 자체가 변경/삭제된 경우 해당 코멘트 스레드를 자동으로 "Outdated(아웃데이트됨)" 상태로 전환하고 접어 표시한다.

### 2. 대화 스레드 및 토론 관리
- [ ] **CODE-08**: 사용자는 인라인 코멘트에 스레드 형식으로 답글(Reply)을 달 수 있으며, 완료된 토론에 대해 리뷰어 또는 PR 작성자가 "Resolve conversation"을 클릭해 접기 처리(필요 시 재개 가능)할 수 있다.
- [ ] **CODE-09**: 시스템은 Pull Request에 미해결된 코드 리뷰 스레드가 최소 1개 이상 존재할 경우 대상 브랜치로의 안전한 병합(Merge)을 엄격하게 차단한다.

### 3. 리뷰 상태 추적 및 결정
- [ ] **CODE-10**: 사용자는 PR에 대한 리뷰 의견 제출 시 `Comment(단순 피드백)`, `Approve(승인)`, `Request Changes(변경 요청)` 중 하나를 선택할 수 있으며, 이 결과는 PR 페이지 우측 상태창에 실시간으로 표시된다.
- [ ] **CODE-11**: 시스템은 코드 수정 후 신규 커밋이 푸시될 경우 기존에 부여되었던 리뷰 승인(Approve) 상태를 자동으로 취소하고 재검토를 강제한다.
