# Requirements: v1.4 웹훅, LFS, 조직 및 보안 기능 강화

## 1. 보안 및 인증 기능 강화 (Security & Auth Enhancements)

- **소셜 로그인 (OAuth2):**
  - GitHub 및 Google 로그인 연동 지원.
  - 최초 소셜 로그인 시 계정 연동 혹은 신규 가입 프로세스 제공.
- **2단계 인증 (2FA - TOTP):**
  - 사용자 프로필 설정 화면에서 TOTP 기반 2FA 활성화/비활성화 기능.
  - QR 코드를 통한 OTP 앱(Google Authenticator 등) 연동 및 백업 복구 코드 생성.
  - 로그인 시 2FA가 활성화된 계정은 비밀번호 인증 후 OTP 번호 입력을 요구하는 다단계 인증 적용.
- **세션 및 보안 정책 관리:**
  - 사용자 프로필 또는 관리자 설정에서 현재 로그인된 기기 및 활성 세션(Session) 목록 조회 기능.
  - 특정 기기 세션 강제 종료(원격 로그아웃) 기능.
  - 일정 기간 비활동 시 세션 만료 및 재인증 요구 정책 적용.

## 2. 조직 및 팀 기능 (Organizations & Teams)

- **조직 (Organization):**
  - 사용자는 새로운 조직을 생성할 수 있으며, 조직 이름은 URL 경로(`/orgs/{orgname}`)에 고유하게 사용됨.
  - 조직별 대시보드 및 소속 저장소 목록 표시 UI.
- **팀 (Team) 관리:**
  - 조직 내에 여러 개의 팀을 생성 가능 (예: Developers, Designers 등).
  - 팀에 조직원들을 추가/제거할 수 있는 관리 기능.
- **권한 제어 (Access Control):**
  - 조직 소유 저장소에 대해 팀 또는 개별 사용자 단위로 권한 부여.
  - 권한 레벨 세분화: Read (읽기), Write (쓰기 및 푸시), Admin (저장소 설정 관리 및 삭제).

## 3. Git LFS (Large File Storage) 지원

- **Git LFS API 구현:**
  - Git 클라이언트의 LFS 요청을 처리할 수 있는 `objects/batch` 엔드포인트 구현.
  - LFS 잠금 기능 지원을 위한 `objects/locks` 관련 API 엔드포인트 구현.
- **스토리지 백엔드:**
  - LFS 파일(바이너리 데이터)을 저장할 로컬 파일 스토리지 경로 지정 및 보관 로직.
  - LFS 업로드 시 JWT 또는 Basic Auth를 통한 사용자 인증 검증.
- **웹 UI 통합:**
  - 저장소 파일 브라우저에서 LFS 포인터 파일 감지 시, 파일 크기 표시 및 실제 파일 다운로드 링크 제공.
  - (옵션) 이미지, PDF 등 웹 브라우저에서 표시 가능한 LFS 파일의 UI 프리뷰 제공.

## 4. 웹훅 및 외부 연동 (Webhooks & Integrations)

- **웹훅 (Webhooks) 등록 및 관리:**
  - 개별 저장소 설정 페이지에서 웹훅 생성, 수정, 삭제 기능.
  - 이벤트 트리거 대상 설정 (Push, Pull Request, Issue 생성/변경 등).
  - 웹훅 호출을 수신할 외부 대상 URL 및 서명 검증을 위한 Secret Key 설정.
- **배달 이력 및 재시도:**
  - 웹훅 요청 및 응답 본문, 상태 코드 등 상세 배달 기록(Delivery Log) 저장.
  - 웹훅 배달 실패 시 상세 내역 확인 및 웹 UI에서 수동 재전송(Redelivery) 트리거 기능.
- **템플릿 연동:**
  - 슬랙(Slack), 디스코드(Discord) 수신용 웹훅 페이로드 형식에 맞춘 기본 템플릿 연동 지원.
